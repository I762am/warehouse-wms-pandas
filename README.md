# WMS-Модуль адресного хранения и контроля остатков склада

Локальный интерактивный инструмент на Python для автоматизации складской логистики, оптимизации адресного хранения и контроля дефицитных (штучных) позиций.

## 🏢 Бизнес-контекст и решаемая проблема

На крупных торговых и производственных складах (например, склад керамогранита и кафельной плитки) ручной поиск штучного товара в общих таблицах занимает много времени. При постоянном перемещении позиций («сироток», оставшихся по 1-5 шт.) кладовщики часто забывают или ошибаются при ручном переписывании координат в Excel, что приводит к пересортице и потере товара.

**Решение:** Автономный Python-скрипт, который автоматизирует рутину, исключает человеческий фактор при обновлении координат и мгновенно подсвечивает критически низкие остатки.

## 🛠️ Архитектура системы (Data Separation)

Из соображений безопасности и отказоустойчивости система спроектирована по принципу разделения данных (эмуляция реальной интеграции с ERP-системой предприятия):
1. `enterprise_erp.xlsx` — Центральная база данных предприятия (доступ только на чтение). Содержит актуальные остатки и наименования товаров.
2. `warehouse_locations.xlsx` — Логистическая база адресного хранения (доступ на чтение и запись). Содержит координаты: Ворота, Этаж, Ряд.

Скрипт выступает в роли "моста" (интегратора), объединяя данные из двух источников по ключевому полю `Код_Товара`.

## 🚀 Основной функционал

* **Мгновенный адресный поиск:** Объединение данных (Data Joining) через Pandas. По коду товара система выдает полное имя и точные координаты на складе.
* **Умный контроль дефицита:** Если остаток товара на складе составляет **меньше 5 штук**, система автоматически выводит яркое предупреждение в консоль для повышенного внимания кладовщика.
* **Автоматизированное обновление координат:** При перемещении плитки пользователь вводит новые данные в консоль, а скрипт сам находит нужную строку по маске и перезаписывает Excel-файл, исключая риск повредить соседние ячейки.
* **Встроенный Big Data генератор:** Функция стресс-теста, которая за 1 секунду генерирует базу на 10 000 уникальных артикулов для проверки скорости работы алгоритма на больших объемах данных.

## 💻 Технологический стек

* **Python 3** (Базовая логика, циклы, интерактивный CLI-интерфейс)
* **Pandas** (Высокопроизводительный анализ данных, фильтрация, маскирование и слияние таблиц)
* **Openpyxl** (Движок для безопасной записи и сохранения файлов Excel)
* **Colorama** (Визуализация и цветовое кодирование системных предупреждений в консоли)
